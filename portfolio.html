Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - A list of use cases and scenarios for how the elevator may be used with accompanying sequence diagrams.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - Contains requirements for all control modules in the design.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - This file proves traceablity from sequence diagrams to requirements, and from requirements to sequence diagrams.
    • Requirements to Contstraints Traceability - This file proves traceability from requirements to contraints and constraints to requirements.
    • Statecharts to Code Traceability - This file proves traceability from state chart states/transitions to code.
  • Implementation
    • Elevator Control Package - Java implementation of all elevator control modules.
  • Test
    • Unit Test Log - The table describes the unit tests and summarizes the current unit test results.
    • Unit Test Summary File - parseable list of unit test files
    • Integration Test Log - The table describes the integration tests and summarizes the current integration test results.
    • Integration Test Summary File - parseable list of integration test files
    • Acceptance Test Log - The table describes the acceptance tests and summarizes the current acceptance test results.
    • Fault Tolerance Test Log - <description here>
  • Log Files
    • Issue Log - A log of all issues found during development and the steps taken to correct them.
    • Improvements Log - A log of improvement ideas for this course.
    • Peer Review Log - A log of all peer reviews performed throughout the development process.
  • Scheduling
    • Network Schedule - Specifies optimized network message protocol, include prioritization.
  • Verification
    • High Level Requirements Verification Framework - Description of Runtime Monitoring for verification and corresponding logs.
